#+title: docker workbench

* Orbit Emacs VNC Workbench

The Docker workbench runs Orbit Emacs inside a Linux container and exposes the
GUI through VNC on localhost.

This path is experimental.  The preferred work setup is native GUI Emacs with
Dockerized tooling wrappers; see [[file:toolbox.org][dockerized tooling]].  Use this VNC workbench only
when native Emacs is not available or when a fully containerized GUI is worth
the extra display-server complexity.

The container includes:
- Emacs 30.2 by default
- TigerVNC Xvnc, Openbox, and VNC access on localhost
- Git, ripgrep, Universal Ctags, Doxygen, Graphviz, Tcl/Tk, Python, pipx, =tclint=, =tclfmt=, and common build tools
- this Orbit Emacs config with Elpaca packages prewarmed during image build

The VNC server binds to =127.0.0.1= by default through Docker Compose.

* Build

From the config repository:

#+begin_src shell
scripts/orbit-docker build
#+end_src

To build a different Emacs version:

#+begin_src shell
EMACS_VERSION=30.2 scripts/orbit-docker build
#+end_src

To build a different Tcl tooling version:

#+begin_src shell
TCLINT_VERSION=0.8.0 scripts/orbit-docker build
#+end_src

* Start

Mount a project into =/workspace=:

#+begin_src shell
scripts/orbit-docker up /path/to/project
#+end_src

The launcher recreates the container when needed while preserving the named
Docker volumes for packages and user-local configuration.

Then connect with a VNC client:

#+begin_example
localhost:5901
#+end_example

Useful environment overrides:

#+begin_src shell
ORBIT_VNC_PORT=5902 scripts/orbit-docker up /path/to/project
VNC_GEOMETRY=2560x1440 scripts/orbit-docker up /path/to/project
VNC_PASSWORD='change-me' scripts/orbit-docker up /path/to/project
#+end_src

* Keyboard Repeat And Resolution

The container uses TigerVNC's virtual X server instead of mirroring a fixed
Xvfb display.  This matters for two Vim-heavy behaviors:

- held keys such as =j= and =k= repeat normally
- compatible VNC viewers can request real desktop-size changes instead of
  stretching a fixed framebuffer

In RealVNC Viewer, use the toolbar's =1:1= / dynamic-resolution behavior rather
than a scaled view.  When the viewer asks for a new desktop size, TigerVNC
resizes the container display and Emacs gets a real resolution change.

The initial size still comes from =VNC_GEOMETRY=:

#+begin_src shell
VNC_GEOMETRY=2560x1440 scripts/orbit-docker up /path/to/project
#+end_src

Keyboard repeat defaults are tuned for Vim-style movement:

#+begin_src shell
KEY_REPEAT_DELAY=220 KEY_REPEAT_RATE=55 scripts/orbit-docker up /path/to/project
#+end_src

* Runtime State

Docker Compose creates two named volumes:

| Volume             | Container path                    | Purpose                         |
|--------------------+-----------------------------------+---------------------------------|
| =orbit-emacs-var=  | =/home/orbit/.config/emacs/var=   | Elpaca packages and runtime state |
| =orbit-emacs-user= | =/home/orbit/.orbit-emacs.d=      | machine-local =config.el= and snippets |

The image seeds =var/= during build.  On first run, an empty =orbit-emacs-var=
volume is populated from that seed so the container does not have to rebuild
all Emacs packages immediately.

* MIB Roots

Inside the container, configure MIB roots using =/workspace= paths in
=/home/orbit/.orbit-emacs.d/config.el=:

#+begin_src emacs-lisp
(setq orbit-user-mib-roots
      '(("MIB-A" . "/workspace/data/mib-a/")
        ("MIB-B" . "/workspace/data/mib-b/")
        ("MIB-C" . "/workspace/data/mib-c/")
        ("MIB-D" . "/workspace/data/mib-d/")))

(setq orbit-user-mib-icd-version "7.2")
#+end_src

* Tooling

The image installs the common tools Orbit knows how to use from PATH:

#+begin_src emacs-lisp
(setq orbit-user-rg-program "rg")
(setq orbit-user-ctags-program "ctags")
(setq orbit-user-doxygen-program "doxygen")
#+end_src

The image installs =tclint= from PyPI, which also provides =tclfmt=.  If your
workplace provides internal versions, add them in a derived image or mount them
into the container and point Orbit at them:

#+begin_src emacs-lisp
(setq orbit-user-tclint-program "/opt/work-tools/bin/tclint")
(setq orbit-user-tclfmt-program "/opt/work-tools/bin/tclfmt")
#+end_src

* Shell And Logs

Open a shell:

#+begin_src shell
scripts/orbit-docker shell
#+end_src

Follow logs:

#+begin_src shell
scripts/orbit-docker logs
#+end_src

Stop the workbench:

#+begin_src shell
scripts/orbit-docker stop
#+end_src

Inside the container, process logs are written under =/tmp=:
- =/tmp/emacs.log=
- =/tmp/emacs.err=
- =/tmp/vnc.log=
- =/tmp/vnc.err=
- =/tmp/supervisord.log=

* Git Credentials

If you need Git over SSH from inside the container, uncomment the read-only
mounts in =docker-compose.yml=:

#+begin_src yaml
- ${HOME}/.ssh:/home/orbit/.ssh:ro
- ${HOME}/.gitconfig:/home/orbit/.gitconfig:ro
#+end_src

On locked-down machines, HTTPS remotes or project-local credentials may be
easier to manage.
