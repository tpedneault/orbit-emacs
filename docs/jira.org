#+title: jira sync

* Scope

This first Jira pass is read-only:
- fetch issues from Jira Server/Data Center
- sync them into =jira.org=
- open the browser issue from Org

Not included yet:
- comments
- transitions
- worklogs
- write-back from Org to Jira

* Setup

Set these in your user-local =config.el=:

- =orbit-user-jira-base-url=
- =orbit-user-jira-api-prefix=
- =orbit-user-jira-project-key=
- =orbit-user-jira-username=
- =orbit-user-jira-jql=
- =orbit-user-jira-org-file=
- =orbit-user-jira-token-command= or =orbit-user-jira-pat-env=

Defaults:
- API prefix: =/rest/api/2=
- Org file: =jira.org= under your Org directory

Authentication:
- keep the PAT out of the repo
- either fetch it with =orbit-user-jira-token-command=
- or point =orbit-user-jira-pat-env= at an environment variable name

Example:

#+begin_src emacs-lisp
(setq orbit-user-jira-base-url "https://jira.example.com")
(setq orbit-user-jira-project-key "PROJ")
(setq orbit-user-jira-username "thomas")
(setq orbit-user-jira-pat-env "JIRA_PAT")
#+end_src

* JQL

If =orbit-user-jira-jql= is set, orbit-emacs uses it directly.

Otherwise it builds:

#+begin_example
project = <project-key> AND assignee = <username> AND resolution = Unresolved ORDER BY updated DESC
#+end_example

That means v1 sync is intentionally scoped to:
- one configured project
- issues assigned to the configured user

* Commands

| Key         | Action                | Notes                     |
|-------------+-----------------------+---------------------------|
| =SPC n j i= | =mod-jira-import-issue= | import one issue by key or number |
| =SPC n j s= | =mod-jira-sync=       | fetch and refresh =jira.org= |
| =SPC n j r= | =mod-jira-refresh-issue= | refresh current issue subtree only |
| =SPC n j o= | =mod-jira-open-issue= | open issue at point in browser |
| =SPC n j j= | =mod-org-capture-journal= | journal capture moved here |

Import input:
- full key like =PROJ-75=
- bare number like =75=

If you enter a bare number, orbit-emacs prefixes it with =orbit-user-jira-project-key=.

* Org Sync Shape

Each issue becomes one heading in =jira.org=.

Synced headings are agenda-friendly:
- they get a =:JIRA:= tag
- they get an Org TODO keyword mapped from the Jira status
- they keep the original Jira status in =JIRA_STATUS=

Status mapping:
- To Do / Open / Reopened -> =TODO=
- In Progress -> =IN-PROGRESS=
- Blocked / Waiting -> =WAIT=
- Done / Closed / Resolved -> =DONE=
- unknown status -> =TODO=

Heading shape:

#+begin_example
* TODO ABC-123 Summary :JIRA:
#+end_example

Managed content under each issue:

#+begin_example
** Description
** Comments
** Notes
#+end_example

Managed by sync:
- =Description=
- =Comments=

User-editable and preserved across refresh:
- =Notes=

That means you should keep local notes under =** Notes=.
Sync may replace =Description= and =Comments= on every refresh, but it will not overwrite =Notes=.
The single-issue refresh command updates only the issue subtree at point and leaves other Jira issue subtrees unchanged.

Stored properties include:
- =JIRA_KEY=
- =JIRA_STATUS=
- =JIRA_ASSIGNEE=
- =JIRA_PRIORITY=
- =JIRA_UPDATED=
- =JIRA_URL=

Refreshing keeps local note text already written under =** Notes=.
Existing non-Jira tags on a synced heading are preserved, and =:JIRA:= is kept without duplication.

If you have older freeform content directly under the issue heading, orbit-emacs will carry it forward into =Notes= on refresh.

Because =jira.org= lives under your Org directory by default, synced Jira items appear in the agenda automatically.

Why Jira items were easy to miss before:
- on a fresh setup, =jira.org= might not exist yet
- if the file did not exist, Org agenda discovery had nothing to include

orbit-emacs now ensures =jira.org= exists under the Org directory early, so imported or synced Jira items are visible to the agenda/dashboard.

* Troubleshooting

- Missing config:
  - set the Jira base URL and token source
- Missing default JQL inputs:
  - set =orbit-user-jira-jql=
  - or set both =orbit-user-jira-project-key= and =orbit-user-jira-username=
- HTTP error on sync:
  - confirm the base URL, API prefix, PAT, and Jira Server/Data Center endpoint
