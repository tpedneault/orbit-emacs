#+title: tcl workflow

* Tcl Workflow

Orbit Emacs keeps Tcl support pragmatic and toolchain-based.

Tools used:
- =tclint=
- =tclfmt=
- Universal Ctags
- Doxygen 1.8.17 for Tcl docs

There is no LSP layer here.

Editing defaults:
- indentation width defaults to =3=
- fill column defaults to =120=
- fill-column indicator defaults to the global setting
- tabs default to disabled
- whitespace visibility defaults to the global setting
- current-line highlight defaults to the global setting
- all of these can be overridden in user config

* Local Leader

In =tcl-mode=:

| Key         | Action                 | Notes                      |
|-------------+------------------------+----------------------------|
| =SPC m l=   | lint current file      | runs =tclint=              |
| =SPC m f=   | format current file    | runs =tclfmt=              |
| =SPC m TAB= | toggle fold            | manual fold at point       |
| =SPC m r=   | rebuild TAGS           | project-local =TAGS=       |
| =SPC m g=   | jump to definition     | uses project =TAGS=        |
| =SPC m s=   | search Tcl symbols     | project symbols            |
| =SPC m v=   | validate Tcl tooling   | diagnostics report         |
| =SPC m e=   | show tool output       | =*tcl-tools*=              |
| =SPC m d d= | project docs manual    | opens =docs/<project>=     |
| =SPC m d s= | search docs            | Doxygen XML                |
| =SPC m d p= | docs at point          | current symbol             |
| =SPC m d r= | regenerate Doxygen XML | runs Doxygen               |
| =q=         | go back                | Tcl docs buffers only      |

* TAGS Workflow

- Universal Ctags is used for Tcl navigation and proc highlighting.
- The =TAGS= file lives at the project root.
- =SPC m r= rebuilds the project-local =TAGS= file.
- =SPC m g= visits the project =TAGS= file automatically and jumps to the definition at point.
- =SPC m s= searches known Tcl symbols gathered from project =TAGS= and optional user-known symbols.
- Rebuild =TAGS= when navigation or highlighting looks stale.

* Namespace Behavior

Orbit Emacs synthesizes fully qualified names from =namespace eval= blocks when
the TAGS data is clear enough to do so.

Example source:

#+begin_src tcl
namespace eval ait::session {
    proc current_step {} { ... }
}
#+end_src

This becomes searchable and highlightable as:

#+begin_example
ait::session::current_step
#+end_example

Practical behavior:
- fully qualified names are preferred in symbol search
- duplicate short-name entries are hidden from symbol search when a qualified form exists
- top-level procs such as =assert= or =titled= still appear normally

* Symbol Highlighting

- project proc highlighting comes from =TAGS=
- qualified proc names such as =ait::log::value= highlight correctly
- synthesized namespace-qualified names can also highlight when the TAGS section is clear enough
- ambiguous short names may intentionally not highlight
- rebuild =TAGS= with =SPC m r= to refresh project proc highlighting

Optional extra symbols:
- set =orbit-user-tcl-known-symbols-file= to a plain text file with one symbol per line
- blank lines are ignored
- lines starting with =#= are ignored
- these symbols are merged into highlighting and symbol search

User config can also override Tcl editing defaults with:
- =orbit-user-enable-fill-column-indicator=
- =orbit-user-fill-column=
- =orbit-user-tcl-indent-width=
- =orbit-user-tcl-fill-column=
- =orbit-user-tcl-use-tabs=
- =orbit-user-tcl-enable-fill-column-indicator= where ='inherit= falls back to the global setting
- =orbit-user-enable-whitespace=
- =orbit-user-enable-hl-line=
- =orbit-user-tcl-enable-whitespace= where ='inherit= falls back to the global setting
- =orbit-user-tcl-enable-hl-line= where ='inherit= falls back to the global setting

The Tcl tooling diagnostic command =SPC m v= reports:
- =tclint=
- =tclfmt=
- resolved =ctags= path, including an Apple ctags warning when detected
- Doxygen program
- Doxygen config file
- Doxygen XML directory and =index.xml=

* Tcl Docs

Tcl docs use Doxygen XML and open in temporary read-only Org buffers.

Commands:
- =SPC m d d= opens the project docs manual
- =SPC m d s= searches indexed docs symbols
- =SPC m d p= opens docs for the symbol at point
- =SPC m d r= regenerates the XML with Doxygen

Behavior:
- docs open in =docs/<project>=
- source links open in =edit/<project>=
- =q= goes back inside Tcl docs buffers

Defaults:
- =<project-root>/Doxyfile=
- =<project-root>/docs/xml/=

User config can override:
- =orbit-user-doxygen-program=
- =orbit-user-doxygen-config-file=
- =orbit-user-tcl-doxygen-xml-directory=

If the XML directory or =index.xml= is missing, regenerate docs with =SPC m d r=.

* Folding

- folding is manual and powered by =vimish-fold=
- use visual selection plus =zf= to create a fold around a verbose Doxygen block
- =SPC m TAB= or =za= toggles a fold at point
- =zo= opens a fold
- =zc= closes a fold
- =zd= deletes a fold

Why manual only:
- the earlier comment-scanning fold experiment was intentionally removed
- no fold-all comment scanner is used now because stability matters more than automation

* Doxyfile Requirements

Minimum settings:
- =GENERATE_XML = YES=
- =XML_OUTPUT = xml=
- =OUTPUT_DIRECTORY = docs=
- =RECURSIVE = YES=
- =INPUT= should include the Tcl source directory or project root
- =FILE_PATTERNS= should include:
  - =*.tcl=
  - =*.tm=
  - =*.sdc=
  - =*.xdc=
  - =*.upf=

Do not force Tcl through the C parser with =EXTENSION_MAPPING = tcl=C=.
Doxygen 1.8.17 has native Tcl support, and Tcl-to-C extension mapping causes
missing symbols in the XML output.

* Troubleshooting

- =SPC m g= cannot find a symbol:
  rebuild =TAGS= with =SPC m r=
- symbol search is missing entries:
  check the project root and confirm the right =TAGS= file was rebuilt
- docs search is empty:
  regenerate Doxygen XML with =SPC m d r=
- Doxygen misses Tcl symbols:
  verify the =Doxyfile= and avoid =EXTENSION_MAPPING = tcl=C=
- wrong ctags is used on macOS:
  set =orbit-user-ctags-program= to the Homebrew Universal Ctags path

* Typical Loop

1. Open a Tcl project with =SPC x e=.
2. Rebuild =TAGS= with =SPC m r=.
3. Jump with =SPC m g= or search with =SPC m s=.
4. Lint with =SPC m l=.
5. Format with =SPC m f= when needed.
6. Revisit command output with =SPC m e=.
7. Open the docs manual with =SPC m d d= when you want project docs.

* Test Steps

1. Open a Tcl project and run =SPC m r=.
2. Put point on a known proc and use =SPC m g=.
3. Run =SPC m s= and confirm qualified names such as =ait::session::current_step= appear once.
4. Confirm top-level names such as =assert= still appear in symbol search.
5. Run =SPC m l= and =SPC m f= on the current file.
6. Run =SPC m d r=, then use =SPC m d s= or =SPC m d p=.
