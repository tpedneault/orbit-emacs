#+title: tcl workflow

* Tcl Workflow

Orbit Emacs keeps Tcl support pragmatic and toolchain-based.

Tools used:
- =tclint=
- =tclfmt=
- Universal Ctags
- Doxygen 1.8.17 for Tcl docs

There is no LSP layer here.

Editing defaults:
- indentation width defaults to =3=
- fill column defaults to =120=
- fill-column indicator defaults to the global setting
- tabs default to disabled
- whitespace visibility defaults to the global setting
- current-line highlight defaults to the global setting
- all of these can be overridden in user config

* Local Leader

In =tcl-mode=:

| Key         | Action                 | Notes                      |
|-------------+------------------------+----------------------------|
| =SPC m l=   | lint current file      | runs =tclint=              |
| =SPC m f=   | format current file    | runs =tclfmt=              |
| =SPC m F=   | fold definitions       | top-level proc/namespace   |
| =SPC m D=   | fold doxygen comments  | all =##= comment blocks    |
| =SPC m TAB= | toggle fold            | fold at point              |
| =SPC m r r= | rename local variable  | proc-body Tcl refactor     |
| =SPC m r t= | rebuild TAGS           | project-local =TAGS=       |
| =SPC m g=   | jump to definition     | uses project =TAGS=        |
| =SPC m s=   | search Tcl symbols     | project symbols            |
| =SPC m v=   | validate Tcl tooling   | diagnostics report         |
| =SPC m e=   | show tool output       | =*tcl-tools*=              |
| =SPC m d d= | project docs manual    | opens =docs/<project>=     |
| =SPC m d s= | search docs            | Doxygen XML                |
| =SPC m d p= | docs at point          | current symbol             |
| =SPC m d r= | regenerate Doxygen XML | runs Doxygen               |
| =q=         | go back                | Tcl docs buffers only      |

* TAGS Workflow

- Universal Ctags is used for Tcl navigation and proc highlighting.
- The =TAGS= file lives at the project root.
- =SPC m r t= rebuilds the project-local =TAGS= file.
- =SPC m g= visits the project =TAGS= file automatically and jumps to the definition at point.
- =SPC m s= searches known Tcl symbols gathered from project =TAGS= and optional user-known symbols.
- completion-at-point / Corfu uses the same symbol pool, even when only the optional user-known symbols file is available
- when Doxygen XML is available, Corfu also shows compact inline Tcl docs such as args or kind
- documented Tcl symbols also get a small completion margin marker, so functions and other symbol kinds are easier to scan visually
- selected Corfu candidates automatically show a richer right-side popup with summary, details, params, returns, and source when Doxygen XML has that metadata
- completing a documented Tcl proc expands into snippet placeholders for its expected parameters when Yasnippet is active
- Rebuild =TAGS= with =SPC m r t= when navigation or highlighting looks stale.

* Namespace Behavior

Orbit Emacs synthesizes fully qualified names from =namespace eval= blocks when
the TAGS data is clear enough to do so.

Example source:

#+begin_src tcl
namespace eval ait::session {
    proc current_step {} { ... }
}
#+end_src

This becomes searchable and highlightable as:

#+begin_example
ait::session::current_step
#+end_example

Practical behavior:
- fully qualified names are preferred in symbol search
- fully qualified names are also preferred in Tcl symbol completion
- duplicate short-name entries are hidden from symbol search when a qualified form exists
- top-level procs such as =assert= or =titled= still appear normally

* Symbol Highlighting

- project proc highlighting comes from =TAGS=
- qualified proc names such as =ait::log::value= highlight correctly
- synthesized namespace-qualified names can also highlight when the TAGS section is clear enough
- ambiguous short names may intentionally not highlight
- rebuild =TAGS= with =SPC m r t= to refresh project proc highlighting

Optional extra symbols:
- set =orbit-user-tcl-known-symbols-file= to a plain text file with one symbol per line
- blank lines are ignored
- lines starting with =#= are ignored
- these symbols are merged into highlighting, symbol search, and Tcl symbol completion
- the configured path is normalized before loading, so =~/= style user paths work predictably
- a fake test file lives at [docs/examples/tcl-known-symbols-example.txt](/Users/thomas/.config/emacs/docs/examples/tcl-known-symbols-example.txt)
- point your user config at it with:

#+begin_src emacs-lisp
(setq orbit-user-tcl-known-symbols-file
      "/Users/thomas/.config/emacs/docs/examples/tcl-known-symbols-example.txt")
#+end_src

- replace or remove that example file later when you want to use real external symbols

User config can also override Tcl editing defaults with:
- =orbit-user-enable-fill-column-indicator=
- =orbit-user-fill-column=
- =orbit-user-tcl-indent-width=
- =orbit-user-tcl-fill-column=
- =orbit-user-tcl-use-tabs=
- =orbit-user-tcl-enable-fill-column-indicator= where ='inherit= falls back to the global setting
- =orbit-user-enable-whitespace=
- =orbit-user-enable-hl-line=
- =orbit-user-tcl-enable-whitespace= where ='inherit= falls back to the global setting
- =orbit-user-tcl-enable-hl-line= where ='inherit= falls back to the global setting
- =orbit-user-tcl-auto-fold-definitions= to fold top-level definitions on buffer open
- =orbit-user-tcl-auto-fold-doxygen-comments= to fold Doxygen comment blocks on buffer open

The Tcl tooling diagnostic command =SPC m v= reports:
- =tclint=
- =tclfmt=
- resolved =ctags= path, including an Apple ctags warning when detected
- Doxygen program
- Doxygen config file
- Doxygen XML directory and =index.xml=
- configured known-symbols file status and symbol count

* Local Variable Rename

=SPC m r r= renames a Tcl local variable only inside the surrounding =proc=
body. This is a lightweight Tcl-specific refactor helper; it does not use LSP
or tree-sitter.

Supported forms:
- bare variable names in common command positions:
  - =set oldName ...=
  - =incr oldName=
  - =lappend oldName ...=
  - =append oldName ...=
  - =foreach oldName ...=
- variable references:
  - =$oldName=
  - =${oldName}=
  - =$oldName(index)=

Behavior:
- point can be on =oldName=, =$oldName=, =${oldName}=, or =$oldName(index)=
- the command normalizes that to the bare variable name
- when inside a proc, replacements are limited to that proc body
- when not inside a proc (top-level scripts, sourced files), replacements cover the entire buffer
- the confirmation prompt always shows the active scope: =proc NAME= or =entire buffer=
- confirmation happens before editing
- the edit is grouped as one undo step

Limitations:
- this is intentionally conservative
- when scoped to a proc, it does not rename outside the detected =proc=
- it does not currently rename comments
- bare-name renames are only recognized in the common command forms listed above
- bare names inside arbitrary strings are not renamed
- dollar-style references inside strings are renamed because they are clear Tcl variable references

* Tcl Docs

Tcl docs use Doxygen XML and open in temporary read-only Org buffers.

Commands:
- =SPC m d d= opens the project docs manual
- =SPC m d s= searches indexed docs symbols
- =SPC m d p= opens docs for the symbol at point
- =SPC m d r= regenerates the XML with Doxygen

Behavior:
- docs open in =docs/<project>=
- source links open in =edit/<project>=
- =q= goes back inside Tcl docs buffers
- Doxygen XML also enriches Tcl Corfu completion when available
- undocumented symbols still complete normally from =TAGS= and external known symbols
- stale completion docs usually mean the XML needs regeneration with =SPC m d r=

Defaults:
- =<project-root>/Doxyfile=
- =<project-root>/docs/xml/=

User config can override:
- =orbit-user-doxygen-program=
- =orbit-user-doxygen-config-file=
- =orbit-user-tcl-doxygen-xml-directory=

If the XML directory or =index.xml= is missing, regenerate docs with =SPC m d r=.

* Folding

Folding is powered by =hs-minor-mode= (hideshow), which is enabled automatically in every Tcl buffer.

** Definition folding

- =SPC m F= folds all top-level =proc= and =namespace eval= blocks in the buffer
- =SPC m TAB= toggles the hideshow fold at point
- set =orbit-user-tcl-auto-fold-definitions t= to fold definitions automatically when a file opens

Definition folding limits:
- only top-level =proc NAME ARGS {= and =namespace eval NAME {= lines are considered
- control blocks like =if=, =foreach=, =while=, and =switch= are ignored
- if a block cannot be matched safely with balanced-brace movement, it is skipped

** Doxygen comment folding

- =SPC m D= folds all Doxygen comment blocks in the buffer
- set =orbit-user-tcl-auto-fold-doxygen-comments t= to fold them automatically when a file opens

A Doxygen block is a run of consecutive comment lines where the first line starts with =##= (the standard Doxygen marker for Tcl).  Subsequent lines start with =#=.  Example:

#+begin_src tcl
## Return the current session step.
## @param session The session handle.
## @return Step identifier string.
proc current_step {session} {
    ...
}
#+end_src

The =##= line triggers detection; the block ends at the first non-comment line.  Blocks already folded are skipped.

* Doxyfile Requirements

Minimum settings:
- =GENERATE_XML = YES=
- =XML_OUTPUT = xml=
- =OUTPUT_DIRECTORY = docs=
- =RECURSIVE = YES=
- =INPUT= should include the Tcl source directory or project root
- =FILE_PATTERNS= should include:
  - =*.tcl=
  - =*.tm=
  - =*.sdc=
  - =*.xdc=
  - =*.upf=

Do not force Tcl through the C parser with =EXTENSION_MAPPING = tcl=C=.
Doxygen 1.8.17 has native Tcl support, and Tcl-to-C extension mapping causes
missing symbols in the XML output.

* Troubleshooting

- =SPC m g= cannot find a symbol:
  rebuild =TAGS= with =SPC m r=
- symbol search is missing entries:
  check the project root and confirm the right =TAGS= file was rebuilt
- docs search is empty:
  regenerate Doxygen XML with =SPC m d r=
- Doxygen misses Tcl symbols:
  verify the =Doxyfile= and avoid =EXTENSION_MAPPING = tcl=C=
- wrong ctags is used on macOS:
  set =orbit-user-ctags-program= to the Homebrew Universal Ctags path

* Typical Loop

1. Open a Tcl project with =SPC x e=.
2. Rebuild =TAGS= with =SPC m r=.
3. Jump with =SPC m g= or search with =SPC m s=.
4. Lint with =SPC m l=.
5. Format with =SPC m f= when needed.
6. Revisit command output with =SPC m e=.
7. Open the docs manual with =SPC m d d= when you want project docs.

* Test Steps

1. Open a Tcl project and run =SPC m r t=.
2. Put point on a known proc and use =SPC m g=.
3. Run =SPC m s= and confirm qualified names such as =ait::session::current_step= appear once.
4. Confirm top-level names such as =assert= still appear in symbol search.
5. In a proc body, put point on =oldName= or =$oldName= and run =SPC m r r=.
6. Confirm =set oldName=, =$oldName=, =${oldName}=, and =$oldName(index)= rename only inside that proc.
7. Run =SPC m l= and =SPC m f= on the current file.
8. Run =SPC m d r=, then use =SPC m d s= or =SPC m d p=.
