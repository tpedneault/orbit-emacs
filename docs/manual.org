#+title: orbit-emacs manual

* orbit-emacs

Handcrafted Emacs config for modal, keyboard-first work.

This manual is the in-Emacs documentation hub. Use it as the first stop for lookup and daily workflow reference.

* Quick Start

- =SPC SPC=: find a file in the current project, or =find-file= outside a project
- =SPC .=: switch buffers
- =SPC /=: search the current project
- =SPC v=: expand region
- =SPC x e=: jump into an editing context
- =SPC x a=: open the agenda dashboard context
- =SPC n t=: capture an inbox task
- =SPC h d=: reopen this manual read-only
- =RET=: follow an Org doc link in normal state
- =SPC h b= / =SPC h B=: move back and forward through doc links

* System

- Modal: Evil-first editing is the default.
- Context-based: work lives in named workspaces like =edit/<project>=, =git/<project>=, =notes=, and =agenda=.
- Minimal UI: built-in theme, built-in modeline, a lightweight home dashboard, and no sidebar.
- Keyboard-driven: small command language under =SPC= and =SPC m=.

* Main Areas

- Contexts/workspaces: Perspective-backed task separation.
- Org workflow: inbox capture, refile, notes, journal, and a custom work dashboard.
- Vim layer: Evil, surround, commentary, expand-region, argument text objects, and multiple cursors.
- Utility bay: one reusable bottom window for shell, messages, help, and compilation.
- Editing helpers: smart paired delimiters and quiet symbol-at-point highlighting in programming buffers.

* Python Quickstart

- Open a Python project with =SPC x e= or =SPC p p=.
- For uv projects, create the local environment with =uv sync=.
- Install the usual Python dev tools into that environment:
  - =uv add --dev basedpyright debugpy ruff=
- Open a Python file and let Eglot attach.
- Use:
  - =SPC m f b= to format
  - =SPC m g d= / =SPC m g r= to navigate
  - =SPC m r f= to run the current file
  - =SPC m r m= to run a module
  - =SPC m d d= to start debugging
- During an active debug session, prefer the short keys:
  - =b= breakpoint
  - =n= next
  - =i= step in
  - =o= step out
  - =c= continue
  - =p= pause
  - =r= restart
  - =q= quit
  - =R= repl
  - =I= info
- See [[file:python.org][python workflow]] for the full uv, Eglot, Dape, and launch.json translation notes.

* Editing Basics

- =smartparens= is enabled in programming buffers and Org buffers.
- Pair insertion stays conservative and uses the package defaults for quotes, brackets, braces, and parens.
- =show-smartparens-mode= gives delimiter matching feedback while you edit.
- =SPC v= grows the current selection with =expand-region=.
- In programming buffers, occurrences of the symbol at point are highlighted after a short idle delay.
- =evil-surround= and =evil-commentary= are part of the default Vim layer.

Useful examples:
- =ysiw"=: wrap the word at point in quotes
- =cs"'=: change double quotes to single quotes
- =gcc=: comment the current line
- visual =gc=: comment the current selection

* User-Local Layer

orbit-emacs creates a user-local directory parallel to the active Emacs config directory:
- =.orbit-emacs.d/=

It contains:
- =config.el=
- =snippets/=

Use it for machine-specific or personal overrides.  Open it with =SPC f c u= and uncomment any setting you want to change.  All settings are documented with comments inside the file.

Settings available by section:

*General*
- =orbit-user-shell= — shell binary (e.g. =/bin/zsh=)
- =orbit-user-org-directory= — root Org directory
- =orbit-user-snippets-directory= — personal YASnippet snippets

*Fonts*
- =orbit-user-font-family= — monospace font (default and fixed-pitch faces)
- =orbit-user-font-height= — height in 1/10 pt units (140 = 14 pt)
- =orbit-user-font-weight= — weight symbol: =light=, =regular=, =medium=, =semibold=, =bold=
- =orbit-user-variable-pitch-font= — proportional font for org prose (leave nil to stay monospace)
- =orbit-user-variable-pitch-height= — height for variable-pitch; float (=1.0=) or absolute integer
- =orbit-user-variable-pitch-weight= — weight for variable-pitch face

When =orbit-user-font-family= is set, the =fixed-pitch= face is automatically kept in sync, so org tables and code blocks stay aligned even when =mixed-pitch-mode= is active.

*Editor defaults*
- =orbit-user-enable-fill-column-indicator= / =orbit-user-fill-column=
- =orbit-user-enable-whitespace=
- =orbit-user-enable-hl-line=

*Tools* — =orbit-user-rg-program=, =orbit-user-ctags-program=

*TCL* — indent width, fill column, tabs, per-language display overrides, doxygen, linter and formatter paths

*org-roam* — =orbit-user-roam-directory= — roam node directory (defaults to =roam/= inside the org dir)

*Jira* — base URL, project key, username, JQL, org file, PAT source

Useful keys:
- =SPC f c u=: open user config
- =SPC f c U=: open user dir
- =SPC f c S=: open user snippets
- =SPC f c s r=: reload snippets

* Markdown

Orbit Emacs ships =markdown-mode= for editing =.md= and =.markdown= files.
GitHub-Flavored Markdown (=gfm-mode=) is the default for =.md= files.

** Features

- Syntax highlighting, including fenced code blocks rendered in their native mode.
- Header scaling — headings appear at proportional visual sizes.
- URL hiding — link text is shown; the URL is hidden to reduce clutter.
- =pandoc= is used as the preview / export backend when available on PATH;
  falls back to =multimarkdown= or =markdown= automatically.

** Local leader (=SPC m= in markdown buffers)

| Key       | Action                | Notes                                  |
|-----------+-----------------------+----------------------------------------|
| =SPC m p= | preview in browser    | requires =pandoc= or =multimarkdown=   |
| =SPC m e= | export to HTML file   | writes a =.html= file next to the source |
| =SPC m t= | toggle GFM checkbox   | =- [ ]= ↔ =- [x]=                     |
| =SPC m i= | insert image link     |                                        |
| =SPC m l= | insert hyperlink      |                                        |

** External tool

Install =pandoc= for the best preview and export experience:

#+begin_src sh
# macOS
brew install pandoc
# Windows
winget install JohnMacFarlane.Pandoc
#+end_src

Without pandoc, markdown-mode falls back to =multimarkdown= or =markdown= if either is on PATH.  Preview and export still work but output quality may differ.

* Mermaid Diagrams

orbit-emacs ships built-in support for Mermaid diagrams via =mod-mermaid=.

** Requirements

Install the Mermaid CLI before use:

#+begin_src sh
npm install -g @mermaid-js/mermaid-cli
mmdc --version
#+end_src

** Org workflow

Write a diagram inside a src block with a =:file= header arg:

#+begin_example
#+begin_src mermaid :file diagram.svg
graph TD
    A --> B --> C
#+end_src
#+end_example

- =SPC m m e= — execute the block and embed the rendered SVG inline
- =SPC m m p= — render the block at point and open a live side-window PNG preview
- =SPC m m P= — toggle auto-preview mode (re-renders ~1 s after each edit pause)

The live preview uses PNG output for compatibility, while Org block execution can
still write SVG files to disk when you use a =:file diagram.svg= header.

** .mmd workflow

Create or open a =diagram.mmd= file to get =mermaid-mode= with syntax highlighting.

- =SPC m p= — render and open side-window PNG preview
- =SPC m P= — toggle auto-preview mode
- =SPC m e= — compile to a file on disk
- =SPC m o= — open in the browser via mermaid.live

** HTML export

When an Org file containing mermaid src blocks is exported to HTML, the blocks are automatically converted to interactive =<div class="mermaid">= elements backed by the mermaid.js CDN.  No static image files are needed and diagrams render natively in the browser.

** Theme

The default render theme is =redux-color=.  Change it at any time:

#+begin_src emacs-lisp
(setq mod-mermaid-theme "dark")
#+end_src

Valid built-in names: =default=, =forest=, =dark=, =neutral=, =base=, =redux=, =redux-color=, =redux-dark=, =redux-dark-color=.
