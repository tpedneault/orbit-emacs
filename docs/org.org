#+title: org workflow

* Files

Primary Org directory:
- =~/org/=

Core files:
- =inbox.org=
- =tasks.org=
- =projects.org=
- =notes.org=
- =journal.org=

Agenda files are built dynamically from all =.org= files under =~/org/=.

* TODO States

Workflow:
- =TODO=
- =NEXT=
- =IN-PROGRESS=
- =WAIT=
- =DONE=
- =CANCELLED=

Notes:
- =DONE= and =CANCELLED= are terminal.
- State changes are logged with timestamps.
- Logs go into a =LOGBOOK= drawer.
- Moving a task to =WAIT= prompts for a note.

* Clock

Active clock visibility:
- the active Org clock appears in the modeline while a clock is running
- normal Org tasks show the task heading and elapsed time
- Jira issue headings with =JIRA_KEY= show only the key, for example:
  - =[ABC-123] 0:32=

Top-level clock commands:
- =SPC n c g=: jump to the active clock
- =SPC n c o=: clock out
- =SPC n c c=: cancel the active clock

Local Org clock commands stay available inside Org buffers:
- =SPC m c i=: clock in
- =SPC m c o=: clock out
- =SPC m c c=: cancel clock
- =SPC m c g=: jump to active clock

Jira sync:
- =SPC n j c=: add a Jira comment explicitly
- =SPC n j i=: import one Jira issue by key or number
- =SPC n j s=: fetch Jira issues into =jira.org=
- =SPC n j r=: refresh only the current Jira issue subtree
- =SPC n j t=: transition the current Jira issue explicitly
- =SPC n j o=: open the Jira issue at point in the browser
- =SPC n j w=: submit one Jira worklog from an Org CLOCK entry
- =SPC n j W=: submit one manual Jira worklog

Templates:
- inbox task
- quick note
- journal entry

Example:
1. Press =SPC n t=
2. Write the task
3. Finish with =C-c C-c=
4. Refile later with =SPC m r=

* Refile

Refile targets are headings across Org files under =~/org/=.

Main use:
- capture quickly into inbox
- sort later into =tasks.org=, =projects.org=, or another file

Commands:
- in Org buffer: =SPC m r=
- in agenda: =SPC m r=

* Agenda Dashboard

Main command:
- =SPC n a=

Context entry:
- =SPC x a=

The main dashboard contains:
- =Schedule=
- =In Progress=
- =Next=
- =Waiting=
- =Inbox=
- =Triage=

Meaning:
- =Schedule=: today’s scheduled items and deadlines
- =In Progress=: active work already underway
- =Next=: ready-to-pull tasks
- =Waiting=: blocked or delegated work
- =Inbox=: raw =TODO= items from =inbox.org=
- =Triage=: unscheduled =TODO= items from other Org files

* Agenda Usage

In agenda:
- =j= / =k= move up and down
- =RET= visits the selected item in the =notes= context
- =SPC m c i= clocks into the item at point
- =SPC m c o= clocks out
- =SPC m c c= cancels the active clock
- =SPC m c g= jumps to the active clock
- =SPC m t= changes TODO state
- =SPC m s= schedules
- =SPC m d= sets deadline
- =SPC m r= refiles
- =SPC m a= archives

* Org Buffer Workflow

In a normal Org buffer:
- =RET= follows an Org link in normal state
- =SPC h b= / =SPC h B= move back and forward through documentation links

* Practical Examples

** Capture a task fast

1. =SPC n t=
2. Enter: =Fix Dired refresh issue=
3. =C-c C-c=

** Move it out of inbox later

1. Open inbox or agenda
2. Put point on the task
3. Press =SPC m r=
4. Choose a heading under =tasks.org= or =projects.org=

** Mark work as blocked

1. Put point on a task
2. Press =SPC m t=
3. Change state to =WAIT=
4. Enter the note when prompted

** Review the day

1. =SPC n a=
2. Check =In Progress=, =Next=, and =Waiting=
3. Visit items with =RET=

** Move a subtree

1. Put point on a heading.
2. Press =SPC m J= or =SPC m K=.

** Navigate the manual

1. Open =docs/manual.org=.
2. Press =RET= on a file link.
3. Return with =SPC h b=.

* org-babel — Code Block Execution

Orbit Emacs enables org-babel for three languages:

| Language    | Block header          |
|-------------+-----------------------|
| Emacs Lisp  | =#+begin_src emacs-lisp= |
| Python      | =#+begin_src python=  |
| Shell       | =#+begin_src shell=   |

** Running a block

Place point inside (or on the header line of) a src block and press =C-c C-c=.
A confirmation prompt appears before any block executes — this is intentional
and cannot be skipped.  Confirm with =y=.

Results appear immediately below the block as a =#+RESULTS:= section.

Example:

#+begin_example
#+begin_src python
print("hello from org-babel")
#+end_src

#+RESULTS:
: hello from org-babel
#+end_example

** Shell blocks

Shell blocks use the system shell (=sh= / =bash= / =zsh= depending on PATH):

#+begin_example
#+begin_src shell
ls -1 ~/org/
#+end_src
#+end_example

** Python

Python blocks use =python3= by default.  To use a project virtualenv, activate
it in a shell block before running Python blocks, or set:

#+begin_src emacs-lisp
(setq org-babel-python-command "/path/to/.venv/bin/python")
#+end_src

** Security

=org-confirm-babel-evaluate= is =t=.  This means every block requires
explicit confirmation before executing.  Do not lower this setting when opening
org files from untrusted sources.

* Jira

The Jira integration is read-only in this first pass.

Use it to:
- pull assigned issues from one configured Jira project into =jira.org=
- keep private notes under issue headings
- reopen the browser issue from Org
- add comments, transitions, and worklogs only through explicit confirmed commands

Org clock workflow for Jira worklogs:
1. put point on a Jira issue heading
2. =SPC m c i= to clock in
3. =SPC m c o= to clock out
4. =SPC n j w= to submit that time explicitly

Notes:
- write actions are explicit and confirmed
- Org TODO changes do not automatically push to Jira
