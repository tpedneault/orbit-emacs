#+title: workflow

* Morning Planning

1. Open the dashboard with =SPC n a= or =SPC x a=.
2. Review:
   - =Schedule=
   - =In Progress=
   - =Next=
   - =Waiting=
   - =Inbox=
   - =Triage=
3. Visit an item with =RET=.
   - The agenda context stays stable.
   - The source entry opens in the =notes= context.
4. Refile inbox items with =SPC m r=.
5. Adjust state with =SPC m t=, schedule with =SPC m s=, deadline with =SPC m d=.

* Working Loop

1. Jump into a project editing context with =SPC x e= or =SPC p p=.
   - both routes open or reuse =edit/<project>= instead of showing the project action menu
2. Find files with =SPC .= or =SPC p f=.
   - =SPC SPC= is the quickest file entry point: project file finder in a project, =find-file= otherwise
   - =SPC .= is the quick buffer switcher
3. Search with =SPC /=, =SPC s s=, or =SPC p s=.
   - =SPC s b= searches the current buffer
   - =SPC s f= jumps to files by name
   - =SPC s d= searches the current directory
4. Refactor across the current project with:
   - =SPC s r= for safe regexp query replace
   - =SPC s R= for preview + explicit-confirmation regexp replace-all
   - =SPC p R= when you want the older project replace entry point
   - =SPC s R= prompts for an optional file glob
   - examples: empty, =*.el=, =src/**/*.py=, =tests/**/*.tcl=
   - the replace-all confirmation shows the root, search regexp, replacement text, glob filter, and candidate file count before changing anything
   - if ripgrep finds zero candidate files, the command exits cleanly without editing buffers
5. Use =SPC x f= when you want a dedicated file-management workspace.
   - Dired stays in =files/...=
   - opening a file there moves you into the matching =edit/...= context
   - Org-roam files are treated specially and open in =edit/org-roam=
6. Use =SPC x g= or =SPC g g= for Git work.
   - first =SPC x g= initializes the Git context
   - later calls just switch back to it
7. Inspect a file from another revision with =SPC g f f= when you want to compare without touching the working tree.
8. Use the utility bay for transient support:
   - =SPC o s= eat shell (opens or reuses the default shell)
   - =SPC o S= new named eat shell
   - =SPC o m= messages
   - =SPC o c= compilation
7. For small editing moves:
   - =SPC v= expands the current region
   - programming buffers highlight the symbol at point after a short idle pause
   - paired delimiters are handled by =smartparens=
8. Capture interruptions quickly:
   - =SPC n t= inbox task
   - =SPC n N= quick note
   - =SPC n j j= journal entry

* Jira Loop

1. Configure your Jira base URL, auth token source, project key, and username in the user-local config.
2. Open =jira.org= with =SPC n j J= when you want the synced issue list directly.
3. Run =SPC n j s= to sync assigned Jira issues into =jira.org=.
4. Add your own notes under =** Notes= inside an issue heading.
5. Refresh again with =SPC n j s= when Jira changes.
   - notes already written under a synced issue heading are preserved
6. Use =SPC n j o= from a Jira heading to reopen the browser issue.
7. Log time either by:
   - clocking with =SPC m c i= / =SPC m c o= and submitting with =SPC n j w=
   - or using =SPC n j W= for a manual worklog entry

* Roam Loop

1. Open notes with =SPC n r f= or create one with =SPC n r c=.
2. Org-roam files open in =edit/org-roam= even if the roam directory lives inside another project tree.
3. Use that context as the dedicated space for linked-note editing instead of mixing roam files into =edit/<project>=.

* Tcl Loop

1. Open the project in an editing context with =SPC x e=.
2. Rebuild Tcl =TAGS= with =SPC m r t= after symbol changes.
3. Jump with =SPC m g= or search with =SPC m s=.
4. Use =SPC m r r= for proc-local variable rename when you want a conservative Tcl refactor.
5. Lint with =SPC m l= and format with =SPC m f=.

* Python Loop

1. Open a Python project with =SPC x e= or =SPC p p=.
2. For uv projects, run =uv sync= so the project has =.venv/=.
3. Install Python dev tools into that environment:
   - =uv add --dev basedpyright=
   - =uv add --dev debugpy=
   - =uv add --dev ruff=
4. Open a Python file and let Eglot attach when =basedpyright= or =pyright= is available.
5. Navigate with =SPC m g d= and =SPC m g r=.
6. Rename with =SPC m e r= and view docs with =SPC m e h=.
7. Format with =SPC m f b=.
8. Run the current file with =SPC m r f= or module with =SPC m r m=.
9. Start debugging with =SPC m d d=.
10. After the session starts, prefer the short keys:
   - =b= breakpoint
   - =n= next
   - =i= step in
   - =o= step out
   - =c= continue
   - =p= pause
   - =r= restart
   - =q= quit
   - =R= repl
   - =I= info

* Mermaid Loop

** Diagram in an Org file

1. Write a mermaid src block with a =:file= header arg:
   #+begin_example
   #+begin_src mermaid :file diagram.svg
   graph TD
       A --> B
   #+end_src
   #+end_example
2. Press =SPC m m P= to enable auto-preview — a side window opens on the right.
3. Edit the diagram source freely; the preview updates ~1 s after each typing pause.
4. Press =SPC m m e= (=C-c C-c=) to execute the block and embed the rendered SVG inline in the Org buffer.
5. Toggle inline images with =C-c C-x C-v= to see the result without leaving the buffer.

** Standalone .mmd file

1. Open or create a =diagram.mmd= file.
2. Press =SPC m P= to enable auto-preview.
3. Edit with live preview on the right.
4. Press =SPC m e= to compile a final output file on disk.
5. Press =SPC m o= to open the diagram in the browser via mermaid.live when you want the interactive online editor.

** HTML export

Exporting an Org file with mermaid src blocks to HTML automatically produces interactive diagrams in the browser via mermaid.js — no static images needed, no extra steps.

* Shell Workflow

Orbit Emacs uses =eat= (Emulate A Terminal) as the in-Emacs terminal.  It is
pure Emacs Lisp — no native libraries required — and works on macOS, Linux,
and Windows.

** Basic use

- =SPC o s= opens (or reuses) the default eat shell docked in the utility bay.
- =SPC o S= opens a new named eat shell.  You are prompted for a name;
  the buffer is called =*eat:NAME*=.

** Input modes

Eat starts in Emacs mode (normal key behavior).  Two input modes are available:

| Key         | Mode             | Effect                                      |
|-------------+------------------+---------------------------------------------|
| =C-c C-j=   | semi-char mode   | most keys go to the terminal                |
| =C-c C-k=   | emacs mode       | back to normal Emacs / Evil key behavior    |

In Evil normal state you can scroll the terminal output freely.
Press =C-c C-j= when you need to type into the shell.

** Shell selection

The shell is chosen from (first match wins):
1. =orbit-user-shell= — set this in your user config
2. On Windows: =pwsh= (PowerShell 7+) or =powershell= (5.1) on PATH
3. On Unix/macOS: =$SHELL= environment variable

Set in user config for Windows:
#+begin_src emacs-lisp
(setq orbit-user-shell "pwsh")
#+end_src

** Multiple shells

Open additional shells with =SPC o S= and give each a distinct name.
The default shell (=*eat*=) is always available via =SPC o s=.

* Suggested Context Pattern

- =agenda=: planning and review
- =notes=: note-taking and Org item inspection
- =edit/<project>=: coding
- =git/<project>=: repo review and commits
- =files/<project>=: deliberate file operations
- =scratch=: temporary experiments

* End of Day

1. Return to =SPC x a=.
2. Review remaining =IN-PROGRESS=, =NEXT=, and =WAIT= items.
3. Refile loose capture items with =SPC m r=.
4. Add a journal note with =SPC n j= if useful.
5. Save workspace state with =SPC q s=.

* Small Habits That Help

- Use =SPC p a= to remember a project directory so it appears in later project selection.
- Use =SPC p d= to prune stale projects.
- Use =SPC f c= for config access instead of hunting through the repo.
- Put machine-specific overrides in the user-local =.orbit-emacs.d/= layer, not in the repo config.
- Keep personal snippets in the user-local =snippets/= directory and reload with =SPC f c s r=.
- Use contexts for separation, not just convenience.
- Keep the agenda as the dashboard, not the editing surface.
- Capture quickly, then refile later.
- Use the utility bay for transient buffers instead of splitting the main workspace.
